## Notes

- Fulfilment audits should generally be performed *after* compliance audits pass.
  - it is a HARD REQUIREMENT that you exercise every user story covered
    through the actual user interface (GUI, CLI, API, etc.) where feasible
    - while ideally avoided, this may require human assistance
    - if not possible in some case, this must be indicated in the report
    - it is a CRITICAL FAILURE and PROJECT BLOCKER if this is not done
    - evidence of how each story was audit must be present in the report in
      very specific terms (not just "tests', or similar)
        - this may include text generated by running tests, screenshots, running live demos, etc.
