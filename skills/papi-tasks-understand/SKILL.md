# Tasks - Understand

- Work is nominally planned, performed according to, and tracked by, task documents.
  - Some trivial work may be able to be performed without a task document, but check with the user if in doubt.
- A new task document MUST be created from a relevant template that is the most specific for the task.
  - If no template seems to fit well, check with the user on how to proceed.
    - Suggest a new task template if appropriate.
- Planned, active, completed and abandoned tasks each will have their own locations.
  - As a task changes state, the task document should be moved to one of these locations.
  - If your instructions do not EXPLICITLY include the locations for task docs, inform the user.
- Where a task would take more than one *session*, or is excessively complex, sub-tasks should be created:
  - tasks should continue to be broken down until sizing/complexity for each task is appropriate

## Basic Tasks

- For basic tasks that don't fit into more specific templates:
  - read, copy and use `basic-task-template.md` or
  - suggest creating a new template if appropriate.
